# MyDeepTalk Evolution (v1.1)

## Mission

Version 1.1 extends MyDeepTalk without disrupting the stable platform.

The goal is to introduce intelligent capabilities while preserving existing workflows.

---

## Engineering Principles

- main is always deployable.
- One feature at a time.
- Keep the build green.
- Prefer extension over modification.
- Build engines before UI.
- Business logic belongs in services.
- React components should orchestrate, not contain business logic.
- Every feature should have a single integration point.
- Every commit should be independently reversible.
